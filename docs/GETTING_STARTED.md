# Getting Started

## Prerequisites

- Docker + Docker Compose
- A [Groq API key](https://console.groq.com) (free tier is enough to try this)

## 1. Configure

```bash
cp .env.example .env
```

At minimum, set `SENTINELAI_GROQ_API_KEY` in `.env`. Everything else has a
working default for local Docker Compose.

## 2. Start the stack

```bash
docker compose up -d
```

This starts: `postgres`, `redis` (Redis Stack, for RedisVL's semantic
cache), `chromadb`, `rabbitmq`, `backend` (FastAPI), 2× `worker` (Celery),
`prometheus`, `grafana`.

Check everything's up:

```bash
curl http://localhost:8000/api/v1/health/live
# {"status": "ok"}
```

API docs (auto-generated by FastAPI): http://localhost:8000/docs

## 3. Ingest a document

```bash
curl -X POST http://localhost:8000/api/v1/documents/ingest \
  -H "Content-Type: application/json" \
  -d '{
    "doc_id": "employee-handbook",
    "doc_version": 1,
    "raw_text": "Employees are entitled to 20 days of paid leave per year. Remote work requires manager approval and a signed WFH agreement.",
    "source_metadata": {"department": "HR", "author": "policy-team"}
  }'
```

Response is immediate (`status: "queued"`, plus a `task_id`) — ingestion
runs asynchronously on a Celery worker. Poll status:

```bash
curl http://localhost:8000/api/v1/documents/ingest/<task_id>/status
```

## 4. Ask a governed question

```bash
curl -X POST http://localhost:8000/api/v1/chat/query \
  -H "Content-Type: application/json" \
  -d '{
    "session_id": "demo",
    "question": "How many paid leave days do employees get?"
  }'
```

Response includes not just the answer, but the governance metadata:

```json
{
  "answer": "Employees are entitled to 20 days of paid leave per year. [1]",
  "confidence": 0.94,
  "citation_coverage": 1.0,
  "groundedness_avg": 0.89,
  "citations": [{"index": 1, "chunk_id": "employee-handbook:v1:0", "doc_id": "employee-handbook", "rerank_score": 8.2}],
  "cache_hit": false,
  "model_used": "llama-3.1-8b-instant",
  "latency_ms": 640,
  "estimated_cost_usd": 0.00004,
  "guardrail_flags": [],
  "evaluation": {"faithfulness": 0.89, "context_utilization": 1.0, "context_precision_proxy": 0.89, "flagged_sentence_count": 0}
}
```

Ask the same (or a semantically similar) question again — `cache_hit` flips
to `true` and `latency_ms` drops sharply.

## 5. Observability

- Prometheus: http://localhost:9090 — try querying `sentinelai_cache_hits_total`
- Grafana: http://localhost:3001 (login `admin` / `admin`) — point it at the
  Prometheus datasource (`http://prometheus:9090`) and build a hit-ratio
  panel per cache type
- RabbitMQ management UI: http://localhost:15672 (`guest` / `guest`)

## 6. Cache admin

```bash
curl http://localhost:8000/api/v1/cache/stats

curl -X POST http://localhost:8000/api/v1/cache/invalidate/document \
  -H "Content-Type: application/json" -d '{"doc_id": "employee-handbook"}'
```

## Running the backend outside Docker (for active development)

```bash
cd backend
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
export $(cat ../.env | xargs)   # or use python-dotenv
uvicorn app.main:app --reload
```

You'll still need Postgres/Redis/Chroma/RabbitMQ running somewhere
reachable — easiest is `docker compose up -d postgres redis chromadb
rabbitmq` and pointing the env vars at `localhost` instead of the service
names.
